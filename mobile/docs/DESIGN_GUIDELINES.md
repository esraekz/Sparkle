# Design & Development Guidelines

## Platform Guidelines

### iOS (Apple)
- **Human Interface Guidelines**: https://developer.apple.com/design/human-interface-guidelines/
- Key focus areas:
  - Typography and spacing
  - Touch targets (minimum 44x44pt)
  - Safe areas and notch handling
  - iOS-specific navigation patterns
  - Accessibility (VoiceOver, Dynamic Type)

### Android (Google)
- **Material Design 3**: https://m3.material.io/
- Key focus areas:
  - Material components
  - Navigation patterns
  - Adaptive layouts
  - Accessibility standards

### React Native Best Practices
- **React Native Docs**: https://reactnative.dev/docs/getting-started
- **Expo Guidelines**: https://docs.expo.dev/guides/overview/

## App Store Requirements
- **iOS App Store Review Guidelines**: https://developer.apple.com/app-store/review/guidelines/
- **Google Play Policy**: https://play.google.com/about/developer-content-policy/

## Important Notes
- Prioritize native feel over web patterns
- Use platform-specific UI components when appropriate
- Test on multiple screen sizes (iPhone SE to Pro Max, various Android)
- Follow accessibility best practices from day one
